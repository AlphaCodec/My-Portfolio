A client-side encrypted file system explorer. Files are encrypted in-browser with AES-GCM, keys derived through salted, high-iteration PBKDF2, and stored as ciphertext-only in IndexedDB — nothing ever leaves the device unencrypted.

Process

How I actually build.

01

Security is default, not a patch

Web-Based-EFS encrypts before anything touches storage — the pattern I default to whenever a project handles data that isn't mine to see.

02

Split services early

Eventify's frontend and backend live in separate repos on purpose — a boundary that's cheap to draw early and expensive to retrofit later.

03

Ship inside the deadline

Two hackathon builds, two working demos, zero "we'll finish it after." Constraints are a feature of the process, not an excuse.
